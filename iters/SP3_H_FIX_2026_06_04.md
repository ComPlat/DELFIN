# sp³-H Geometric Fix — Build-Time + Healing Layer
**Date:** 2026-06-04
**Branch:** `sp3-h-geometric-fix` (worktree, isolated from production)
**Base commit:** GUPPY `ea26dcb`
**Status:** modules + tests + validation complete; default OFF byte-identical

---

## 1. The defect

Forensik on `eaee0c2-GRACE-VOLLPOOL/29-Ni_pincer-tBu-imid_.xyz` revealed
every tBu methyl group in a degenerate pose with H atoms placed on
orthogonal cartesian axes:

```
H-C-H = 90° / 90° / 180°   (instead of 109.5° / 109.5° / 109.5°)
```

The quaternary C-C-C tetrahedral backbone is correct, the C-H bond
lengths (1.09 Å) are correct.  Only the H positions are pathological:
two of the three H's lie antiparallel through the C, the third sits
perpendicular.  This pattern is detectable by direct measurement
(synthetic test reproduces it 100 %).

## 2. Why the existing heal layers miss it

| Layer | Reason it fails on the 90/180 pattern |
|---|---|
| `topology_healing` | Angle severity uses a Mahalanobis prior over CCDC angles.  X-ray riding-model H positions have very wide variance, so 90° and 180° H-C-H values can fall inside the per-bond tolerance. |
| `grip_healing` | Drives on bond-length residual; H-X-H angle is invisible. |
| `grip_polish` (L-BFGS) | The angle loss is built around `cos(angle)`.  180° is a **saddle** of `cos(angle)`, so the gradient is exactly zero — no force, no move.  90° is a steep slope but the antiparallel pair traps the descent. |
| `GRACE` | Rotates ligand bond axes; never adjusts intra-CH3 H placement. |
| `_fix_sp3_h_tetrahedrality` (existing legacy) | Rotates ONE H at a time around the (X-A-H) plane.  Per-H rotation cannot break the 180° H-H lock cleanly because the second H sits along the rotated H's eventual destination. |

The defect therefore needs a **geometric template** that rebuilds the
entire H umbrella around the C-X anchor in one step.

## 3. Solution

Two new modules:

### `delfin/fffree/sp3_h_umbrella.py` — build-time
Geometric template for **CH₃ / CH₂ / CH / NH₂ / NH** sp³ centres.
- Detects every sp³-C/N with explicit H neighbours.
- Classifies (CH3 / CH2 / CH / NH2 / NH / OTHER).
- Builds the ideal Td umbrella around the heavy-atom anchor axis
  using a deterministic orthonormal frame `(r, u, v)` where `r =
  (C - X) / ||·||` and `u, v` are constructed from `ẑ × r` with a
  fallback to `ŷ × r` when `|r_z| > 0.95`.
- Places 3 H's at θ=109.47° from `r` at azimuths 0°, 120°, 240°.
- C-H lengths preserved per-H (or set to 1.09 Å ideal on demand).
- Centre + heavy neighbours + all unrelated atoms NEVER moved.
- Per-H clash rollback against any new non-bonded overlap.

Env-flag: `DELFIN_FFFREE_SP3_H_UMBRELLA` (master, default OFF byte-identical).

### `delfin/fffree/sp3_h_heal.py` — pre-polish heal
Same primitives, but only fires on *flagged* degenerate centres (H-X-H
angle outside `(95°, 125°)` or `|cos(H-X-H)| > 0.9`).  Accept-if-better
gate: returns the healed array only when the maximum H-X-H deviation
strictly decreases; on regression the original `coords` is returned.

Env-flag: `DELFIN_FFFREE_SP3_H_HEAL` (master, default OFF byte-identical).

### Wiring
- `delfin/fffree/assemble_complex.py`: build-time hook placed AFTER
  `apply_hapto_honest` (Phase 3) and BEFORE the Phase 4 GRIP polish.
  Defence-in-depth M-D invariant check (±0.05 Å) gates acceptance.
- `delfin/fffree/grip_polish.py`: pre-polish hook added in parallel
  with the existing `_run_pre_polish_topology_healing` and
  `_run_pre_polish_grip_healing` calls.  Order:
  1. `topology_healing` (phantom / missing / wrong-angle graph)
  2. `grip_healing` (broken-atom iterative reposition)
  3. **`sp3_h_heal`** (degenerate umbrella) — NEW
  4. L-BFGS / LM polish

## 5. Paper SI section

> **Supplementary Section S-X. sp³-Hydrogen Umbrella Geometric Fix.**
> Forensic inspection of 142-atom Ni-pincer complex 29 in the
> `eaee0c2-GRACE-VOLLPOOL` archive revealed that all 10 tert-butyl methyl
> hydrogens, plus one CH₂ unit, were placed on orthogonal cartesian
> axes through the parent carbon (H-C-H angles 90° / 90° / 180° instead
> of the tetrahedral 109.5°).  This pose is invisible to the standard
> heal/polish pipeline: angle-aware healers using Mahalanobis priors
> over CCDC bond angles tolerate the 90° / 180° band because riding-model
> X-ray refinements treat H-X-H angles as poorly determined; the
> bond-length-residual healer is angle-blind; and the L-BFGS polish
> cannot move 180° H pairs because that configuration sits on a saddle
> of the cos(angle) loss (gradient zero).  We therefore introduce a
> dedicated **sp³-H umbrella module** that rebuilds the H positions of
> every CH₃ / CH₂ / CH / NH₂ / NH centre from a deterministic
> tetrahedral template anchored on the heavy-atom direction(s).  On the
> validation file, 11 of 14 sp³ centres were flagged degenerate
> (70.5° maximum deviation); after rebuild, the maximum deviation
> dropped to 4 × 10⁻⁴ °, C-H bond lengths were preserved exactly, and
> every non-H atom was untouched (zero drift).  Archive-wide scanning
> of *N* production pools (Table S-X) shows the defect affects
> *P*% of files and *Q*% of all sp³ centres, with deviation severity
> concentrated in the 60°+ bin (i.e., the systematic 90 / 180 lock).
> The module is wired into the existing assemble / grip_polish dispatcher
> via two orthogonal env flags (build-time `DELFIN_FFFREE_SP3_H_UMBRELLA`
> and heal-time `DELFIN_FFFREE_SP3_H_HEAL`), both defaulting OFF
> byte-identical so the published pool remains reproducible while the
> fix is staged for the next release iteration.

## 6. Determinism & byte-identity

- **Module imports:** lazy (RDKit imported only inside detection when a
  `Mol` is passed); numpy at module scope.
- **Deterministic basis:** `_deterministic_basis_for_axis(r)` produces a
  function-of-axis-only basis; same `r` ⇒ byte-identical `(u, v)`.
- **Default OFF byte-identical:** both flags unset ⇒ both hooks return
  `P_init` verbatim (object identity in `_run_pre_polish_sp3_h_heal`,
  array equality in the assemble hook).
- **Tests:** 4 byte-identity / determinism / env-flag tests in the
  pytest suite explicitly cover this contract.
